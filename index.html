التنفيذ

خطوات التثبيت

1

تفعيل ميزة الحاويات (Container Mode)

تمكين بيئة تشغيل الحاويات على RouterOS

RouterOS CLI يتطلب إعادة تشغيل 
/system/device-mode/update container=yes
ملاحظة مهمة:
  • قد يطلب منك الراوتر تأكيداً فيزيائياً عن طريق فصل الطاقة وإعادة تشغيلها
  • سيقوم الراوتر بإعادة الإقلاع تلقائياً بعد التفعيل
  • تأكد من حفظ الإعدادات قبل تنفيذ هذا الأمر

التحقق من التفعيل:

/system/device-mode/print

يجب أن تظهر النتيجة: container: yes

2

إعداد الشبكة الافتراضية (VETH & Bridge)

إنشاء كارت شبكة افتراضي وجسر لعزل حركة المرور

1. إنشاء VETH Interface:

/interface veth add name=veth-sasman address=172.17.0.2/24 gateway=172.17.0.1

2. إنشاء Bridge:

/interface bridge add name=bridge-container

3. ربط VETH بالـ Bridge:

/interface bridge port add bridge=bridge-container interface=veth-sasman

التحقق:

/interface veth print
/interface bridge print
/interface bridge port print
3

تعيين عنوان الـ Gateway

إضافة الآي بي الافتراضي على الـ Bridge

/ip address add address=172.17.0.1/24 interface=bridge-container

التحقق:

/ip address print

يجب أن ترى: 172.17.0.1/24 bridge-container

4

إعدادات جدار الحماية (NAT & Port Forwarding)

السماح بالوصول وتوجيه البورتات

1. السماح للحاوية بالوصول إلى الإنترنت (Masquerade):

/ip firewall nat add chain=srcnat src-address=172.17.0.0/24 action=masquerade comment="Container Internet Access"

2. توجيه بورت الإدارة (88 → 80):

/ip firewall nat add chain=dstnat dst-port=88 protocol=tcp action=dst-nat \
  to-addresses=172.17.0.2 to-ports=80 comment="SASMAN Web Panel"

3. توجيه بورتات RADIUS:

# بورت المصادقة (Authentication)
/ip firewall nat add chain=dstnat dst-port=1812 protocol=udp action=dst-nat \
  to-addresses=172.17.0.2 to-ports=1812 comment="RADIUS Auth"

# بورت المحاسبة (Accounting)
/ip firewall nat add chain=dstnat dst-port=1813 protocol=udp action=dst-nat \
  to-addresses=172.17.0.2 to-ports=1813 comment="RADIUS Acct"

4. السماح بـ RADIUS في Firewall (اختياري):

/ip firewall filter add chain=input action=accept protocol=udp \
  dst-port=1812,1813 comment="Allow RADIUS"

التحقق:

/ip firewall nat print
/ip firewall filter print
5

سحب وتشغيل حاوية Sasman Manager

تثبيت النظام على Disk داخلي أو USB خارجي

الطريقة الأولى: Disk الداخلي

1. تعيين مستودع Docker:

/container/config set registry-url=https://registry-1.docker.io

2. سحب وتثبيت الحاوية:

/container/add remote-image=ahmedkin99/sasman-manager:v5 \
  interface=veth-sasman \
  root-dir=disk1/sasman-data \
  start-on-boot=yes \
  logging=yes

الطريقة الثانية: USB خارجي

1. تهيئة USB:

/disk format usb1 file-system=ext4 label=usb-sasman mbr-partition-table=no

⚠️ سيحذف كل البيانات على الـ USB!

2. إعداد Registry:

/container/config/set registry-url=https://registry-1.docker.io tmpdir=usb1/pull-tmp

3. إنشاء الحاوية:

/container/add name=sasman-manager \
  remote-image=ahmedkin99/sasman-manager:v5 \
  interface=veth-sasman \
  root-dir=usb1/sasman-data \
  logging=yes \
  start-on-boot=yes

مراقبة التثبيت:

/container/print

انتظر حتى تصبح الحالة: status: running

التأكد

التحقق من التثبيت

1. حالة الحاوية

/container/print
النتيجة المتوقعة:
name="sasman-manager"
status: running
remote-image: ahmedkin99/sasman-manager:v5

2. الشبكة

/ping 172.17.0.2 count=5

يجب أن تكون الاستجابة ناجحة من الراوتر إلى الحاوية

3. البورتات

/ip firewall nat print where comment~"SASMAN|RADIUS"

التحقق من وجود قواعد NAT للبورتات 88, 1812, 1813

الاستخدام

مسارات الوصول إلى لوحات التحكم

لوحة الإدارة الرئيسية

الوصول عبر المتصفح

الرابط:

http://[IP-الراوتر]:88

مثال: http://192.168.1.1:88

معلومات تسجيل الدخول الافتراضية:

اسم المستخدم: admin
كلمة المرور: admin

⚠️ غيّر كلمة المرور فوراً بعد أول تسجيل دخول!

اختبار RADIUS من راوتر آخر

التحقق من عمل خادم RADIUS

إضافة خادم RADIUS:

/radius/add address=[IP-الراوتر-الرئيسي] secret=123456 \
  service=hotspot,ppp,login timeout=3s

تفعيل الخادم:

/radius/enable [find address=[IP-الراوتر-الرئيسي]]

اختبار الاتصال:

/radius/test address=[IP-الراوتر-الرئيسي] secret=123456 service=hotspot
الحلول

استكشاف الأخطاء وإصلاحها

الحاوية لا تعمل (status: stopped)

عرض السجلات:

/log print where topics~"container"

إعادة تشغيل يدوياً:

/container/start [find name=sasman-manager]

لا يمكن الوصول إلى البورت 88

التحقق من NAT:

/ip firewall nat print where dst-port=88

التحقق من ping:

/ping 172.17.0.2

التحقق من Firewall:

/ip firewall filter print where chain=forward

RADIUS لا يستجيب

التحقق من UDP Ports:

/ip firewall nat print where dst-port~"1812|1813"

التحقق من الاتصال:

/tool/netwatch add host=172.17.0.2 interval=10s

اختبار RADIUS:

/radius/test address=[IP] secret=123456 service=hotspot

نفاد مساحة التخزين

فحص المساحة:

/disk print

حذف صور قديمة:

/container/remove [find where status=stopped]

الحاوية لا تبدأ تلقائياً بعد إعادة التشغيل

/container set [find name=sasman-manager] start-on-boot=yes
الإدارة

أوامر إدارة مفيدة

إيقاف الحاوية مؤقتاً

/container/stop [find name=sasman-manager]

إعادة تشغيل

/container/start [find name=sasman-manager]

حذف الحاوية (بدون بيانات)

/container/remove [find name=sasman-manager]

حذف كامل (مع البيانات)

/container/remove [find name=sasman-manager]
/file remove [find name~"sasman-data"]

تحديث الحاوية

# إيقاف القديم
/container/stop [find name=sasman-manager]

# حذف القديم
/container/remove [find name=sasman-manager]

# سحب النسخة الجديدة
/container/add remote-image=ahmedkin99/sasman-manager:v5 \
  interface=veth-sasman \
  root-dir=disk1/sasman-data \
  start-on-boot=yes
الأسئلة

الأسئلة الشائعة

نعم، لكن يجب:
  • إنشاء VETH منفصل لكل حاوية
  • استخدام IPs مختلفة (مثل: 172.17.0.3، 172.17.0.4)
  • تعيين Ports مختلفة في NAT
التأثير ضئيل جداً! الحاوية مبنية على SQLite وتستهلك:
  • RAM: 64 MB فقط
  • مساحة: 200 MB فقط
  • CPU: استهلاك منخفض جداً
/file print detail where name~"sasman-data"

يُنصح بنسخ المجلد بالكامل عبر FTP/SFTP

/ip firewall nat add chain=dstnat dst-address-type=local \
  protocol=tcp dst-port=88 action=dst-nat to-addresses=[IP-الراوتر]

⚠️ تحذير أمني: استخدم HTTPS و Firewall قوي!

على كل راوتر ثانوي: 
/radius/add address=[IP-السيرفر-الرئيسي] secret=123456 \
  service=hotspot,ppp,login

/radius/enable [find address=[IP-السيرفر-الرئيسي]]
